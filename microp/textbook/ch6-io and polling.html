SIMPLE I/O DEVICE DESIGN WITH POLLING

Input: Data is read from an input device (keyboard, mouse, disk etc) into a CPU register by the CPU. In the next stage, CPU generally writes this data to a memory location.
In more advanced computers, input devices write their data directly to memory without CPU intermediation. This is called DMA (direct memory access).

Output: Just the reverse of input

How software sees I/O devices

Software sees I/O devices as a group of memory locations, ie, exactly like memory.
We dont need any new instructions to do IO. Regular instructions to deal with memory (LD and ST) is all that is needed.

Memory map

Memory space = 2 to the address bits
Difference between Input, output and memory locations:
Memory: Whatever CPU writes stays there untill overwritten or power out.
Input: The content of this location is determined by an input device, independently of CPU. CPU can only read these locations.
Output:The content of this location is written by the CPU, and this content is sent to the output device.
Example memory map:
The locations in the range can only be written, but may not be written. If we try to write to these locations we may receive unspecified behaviour. This is because these locations are not memory cells, and hence cannot be used for the storage of information. They are used instead to write information into the device registers. Hence, we can st, but not ld these addresses.

An example unsynchronized circuit, with a program that drives it

The best way to learn about IO is to inspect an example first, then design your own system. Hence, without getting bogged down in theory, We will oresent a circuit that
(1) read the data from a a set of 16 switches and
(2) display it on a set of 16-leds and
(3) goes to step (1)
In this way, when we change any switch, it will immediately be reflected in leds.

Memory map of our circuit The software will see our switch set as a single 16 bit memory location. When kth switch is closed, kth bit in this location becomes 1. If kth switch is not closed, the same kth bit is zero. Similar operations
